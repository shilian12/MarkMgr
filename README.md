协作项目：学生成绩管理系统

项目概述

这是一个综合性的成绩管理系统，可由组内6名同学完成

学生A：项目架构师 & 主菜单模块

职责：
· 设计程序整体架构和函数接口
· 实现主菜单循环和程序流程控制
· 协调各模块间的数据传递

学生B：数据输入与验证模块

职责：
· 设计学生信息输入界面
· 实现成绩数据输入功能
· 添加输入数据验证（0-100分）
· 防止非法字符输入

学生C和D：计算核心模块

职责：
· 实现总分计算函数
· 实现平均分计算函数
· 设计成绩统计功能
· 处理计算精度问题

学生E：等级评定模块

职责：
· 设计等级评定标准
· 实现等级判断函数
· 添加评语生成功能
· 设计等级分布统计

学生F：查询与显示模块

职责：
· 设计成绩查询功能
· 实现结果格式化显示
· 创建成绩报告单
· 设计数据展示界面

项目技术要求

基础功能要求
1. 支持5名学生，每名学生3门课程
2. 完整CRUD操作（增删改查）
3. 错误处理机制
4. 用户友好界面

高级功能要求
1. 成绩排名功能
2. 学科分析报告
3. 数据备份恢复
4. 系统配置自定义

考核标准
1. 功能完整性（30%）：所有基础功能实现
2. 代码质量（25%）：规范、注释、可读性
3. 团队协作（20%）：沟通、配合、问题解决
4. 系统稳定性（15%）：错误处理、边界测试
5. 用户体验（10%）：界面友好、操作便捷

扩展挑战（可选）
· 添加图形界面（如使用EGE库）

最后要求：提交一份需求分析报告，源代码，以及运行界面。
